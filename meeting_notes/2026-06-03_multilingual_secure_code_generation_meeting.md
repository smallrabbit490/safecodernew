# 多语言安全代码生成项目会议记录

会议主题：多语言安全代码生成项目进展、数据集翻译情况、后续研究方向与投稿计划

记录日期：2026-06-03

## 一、会议结论

本次会议主要讨论了多语言安全代码生成项目的当前进展、数据集翻译情况，以及后续的研究方向、方法论设计和投稿计划。

总体结论如下：

- C++、Go、Java、JavaScript 等语言的数据集翻译工作已基本完成，仅剩少量未处理条目。
- 后续需要优先完成全部数据集翻译，再安排人工二次校验。
- 研究方向初步确定为：探索一套可应用于多种编程语言的安全代码生成与迁移方法。
- 方法设计可参考生成、测试、分析、修复循环，并考虑加入大模型自进化机制，例如威胁记忆和修复记忆。
- 投稿目标优先考虑 ICSE，如时间紧张则考虑 FSE。

## 二、数据集翻译进展与后续安排

### 1. 翻译进度

当前 C++、Go、Java、JavaScript 等语言的数据集翻译工作已经基本完成。

其中，Java 语言因为部分运行环境需要依赖 Linux，存在一些翻译或验证失败的情况。

### 2. 后续安排

为保证项目整体进度，当前阶段决定先完成所有语言的数据集翻译工作。

翻译完成后，下周安排大二同学协助进行代码二次检查。二次检查的作用是人工确认翻译结果是否真的符合原始 Python 代码的功能语义和安全/不安全语义。

## 三、研究方向与方法论探讨

### 1. 核心目标

项目的核心目标是探索一套能够应用于多种编程语言的安全代码生成方法。

后续研究成果计划投稿至 ICSE 会议。

### 2. 研究思路

会议讨论了将 Python 作为源语言或主流语言的研究路径。

重点问题是：Python 中已有的安全经验、安全代码模式和不安全代码模式，是否可以迁移到 Go、JavaScript 等相对稀缺语言，或 Java 等主流语言中。

这个方向可以理解为：从一个安全数据更丰富的语言出发，帮助其他语言构造安全代码和不安全代码数据集，并通过测试与修复保证迁移质量。

### 3. 方法论

团队参考了 TraceCoder 等项目，讨论了通过以下循环来构建方法：

1. 生成：由大模型生成或翻译目标语言代码。
2. 测试：运行功能测试和安全相关测试。
3. 分析：分析编译错误、运行错误、安全行为不一致等问题。
4. 修复：让大模型根据上一轮错误反馈进行修复。

同时，会议讨论了引入自进化机制的可能性。

这里的自进化可以理解为：系统不是只让大模型一次性生成代码，而是把每轮失败和成功经验记录下来，形成威胁记忆、修复记忆等知识，再反过来辅助下一轮生成和修复。
